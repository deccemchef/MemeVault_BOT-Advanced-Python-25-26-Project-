# Достижения участников команды

## Гребенщикова Варвара / varenik17

**Роль:** Fullstack  
**Зона ответственности:** Клавиатура, база данных, favourites, взаимодействие
пользователя с ботом, разработка команд для бота.

### Реализованные задачи и функциональность

- **Полностью реализована логика взаимодействия с пользователем** -
  наложена логика на машину состояний, позволяет делать общение пользователя с ботом непрерывным, с логичными переходами, написанные функции улучшили исходный функционал взаимодействия, теперь пользователь может в одной "сессии" отправлять n-ое кол-во запросов.
- **Создание и внедрение ReplyKeyboard** — Кнопки "Назад", "Помощь", "Найти
  мем", "Избранное".
- **Создание и внедрение InlineKeyboard** — "Реакции" бота на промежуточные состояния, это позволяет увеличить простоту пользования ботом, но при этом сохранить все исходные функции, также это ускорило общение с ботом.
- **Реализована загрузка мемов (фото/гиф/видео) из админ-канала** - бот берёт подпись, парсит теги и сохраняет мем с тегами в базе, логика отправки запросов совпадает с логикой обработки запросов от пользователей.

### Вклад в командную работу

- Документация: реализация ER-диаграмм.
- Коммуникация: помощь другим с их задачами.

## Воробжанский Егор / deccemchef

**Роль:** Team Lead  
**Зона ответственности:** Пользовательские запросы, поднятие базы данных в
облаке,
взаимодействие бота с пользователем, поднятие бота, введение правил работы с
репозиторием.

### Реализованные задачи и функциональность

- **Введение правил, поднятие бота** - Создание репозитория, обсуждение правил работы с репозиторием, первый запуск бота.
- **Написание машины состояний для поиска мемов по запросам** — бот переводит
  пользователя в режим “ожидания запроса”, принимает текст и
  завершает/сбрасывает состояние.
  Принятый запрос мы разбиваем на несколько подзапросов, для улучшения качества
  поиска.
  **Создание запросов в БД** - Реализована функция, которая принятый запрос разбивает на несколько подзапросов, для улучшения качества поиска.
- **Поднятие базы данных в Яндекс Облаке** — создание кластера, настройка кластера, подключение БД, создание конфига и его внедрение в код, подключение участников к яндекс облаку, распределение ролей. Это позволило нам улучшить качество заполнения бд мемами + упростило обращение с базой данных, потому что теперь нам доступна консоль в WebSQL
- **Безопасность данных** - все секреты помещаются в .env и потом берутся от туда с помощью библиотеки python-dotenv, и читаются через переменные окружения. !.env помещен в .gitignore.
